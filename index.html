⚙️ Model Parameters at a Glance

Temperature (0–2)

Low = focused/factual. High = creative/varied. Default: 0.7

🎯

Top-p (0–1)

Nucleus sampling. 0.9 = top 90% probability tokens only

📏

Max Tokens

Hard cap on output length. 1 token ≈ 0.75 words

🛑

Stop Sequences

Tokens where the model halts generation (e.g. "###")

🔤 Fundamentals
Core concepts every prompt engineer must master, from zero-shot to structured outputs.

🟢 Zero-Shot Prompting

BEGINNER

Ask the model to perform a task with no examples. Works well for simple, well-understood tasks.

# Zero-shot: direct instruction Classify the sentiment of this review as Positive, Negative, or Neutral: "The product arrived fast but the packaging was damaged."
✓ Use whenTask is simple, model knows it well, speed matters
✗ Skip whenTask is niche, nuanced, or output format is specific

🟡 One-Shot & Few-Shot

BEGINNERINTERMEDIATE

Provide 1–5 examples to demonstrate the exact pattern you want the model to replicate.

# Few-shot: show examples first Examples: Input: "I love this!" → Positive Input: "Terrible experience" → Negative Input: "It was okay" → Neutral Now classify: Input: "Better than expected" →
✓ Use whenYou need consistent format or specialized classification
✗ Skip whenExamples don't represent edge cases you care about

🧩 System Prompts

INTERMEDIATE

Set the AI's persistent identity, tone, rules, and capabilities before any user message. The backbone of any deployed app.

SYSTEM: You are Aria, a senior product analyst at a SaaS startup. You: - Always back claims with data or caveats - Never speculate without flagging uncertainty - Respond in concise bullet points max 150 words - Audience: non-technical founders USER: What's our churn risk this quarter?

🎭 Role Prompting

BEGINNERINTERMEDIATE

Assign a specific expert persona to calibrate knowledge depth, vocabulary, and perspective.

# Weak role You are a doctor. Explain heart disease. # Strong role (specific + contextual) You are a board-certified cardiologist with 20 years in preventive medicine. Explain heart disease risk factors to a 35-year-old with no medical background. Use analogies, avoid jargon.

📄 Controlling Output Format

Be explicit. The model will match the format you demonstrate or describe.

JSON OUTPUT
Respond only in valid JSON: { "name": string, "score": 0-10, "reason": string } No extra text.
MARKDOWN TABLE
Return a markdown table with columns: | Feature | Pros | Cons | Include 3-5 rows. Use single-line entries only.
STRUCTURED STEPS
Format your answer as: SUMMARY: [1 sentence] STEPS: 1. [action] 2. [action] CAVEATS: [if any]

📦 Context Window Strategy

The model processes everything in its window equally — position and structure matter.

🔝
Primacy Effect

Instructions at the START of the prompt are followed most reliably. Put rules and persona first.

🔚
Recency Effect

The model also prioritizes what's at the END. Put the actual task/question last for best attention.

📎
Lost in Middle

Long documents stuffed in the middle of prompts are often under-attended to. Chunk or summarize first.

✍️ The 8 Rules of Prompt Clarity

1️⃣

Use imperative verbs

Start with: Write, List, Analyze, Compare, Summarize, Generate, Explain

2️⃣

Specify the audience

"Explain to a 10-year-old" vs "Explain to an ML engineer" — radically different outputs

3️⃣

State the output length

Use word counts or descriptor: "in 3 bullet points", "in under 200 words", "exhaustively"

4️⃣

Positive framing first

"Always use active voice" beats "Don't use passive voice" — tell it what TO do

5️⃣

One task per prompt

Multi-task prompts dilute focus. Chain tasks instead of stacking them in one shot.

6️⃣

Use delimiters

Separate instructions from data with ###, ---, XML tags, or triple quotes

7️⃣

Ground with examples

For complex tasks, show 2–3 input/output pairs before the actual request

8️⃣

Iterate explicitly

Tell the model to critique its first draft: "Now review for errors and rewrite"

🧠 Power Techniques
Advanced prompting methods that dramatically improve reasoning quality and reliability.

🔗 Chain-of-Thought (CoT)

INTERMEDIATE

Force the model to reason step-by-step before giving an answer. Dramatically improves math, logic, and multi-step tasks.

✓ Best forMath, logic puzzles, coding bugs, multi-step analysis
✗ Overkill forSimple lookups, creative one-liners, classification
# Standard CoT trigger Solve this problem. Think step by step before giving your final answer. # Zero-shot CoT magic phrase Let's think through this carefully. # Structured CoT REASONING: [your thought process] ANSWER: [final answer only]

🎯 Self-Consistency Sampling

ADVANCED

Generate multiple outputs with high temperature, then aggregate (majority vote or synthesis). Reduces variance for critical decisions.

# Run 3× with temp=0.8, then: Given these 3 analyses below, identify the consensus conclusion: Analysis 1: {{output_1}} Analysis 2: {{output_2}} Analysis 3: {{output_3}} Synthesize the most reliable answer.

ADVANCED

Problem: {{problem}} Generate 3 different approaches: Approach A: [describe strategy] Approach B: [describe strategy] Approach C: [describe strategy] Evaluate each: pros, cons, feasibility Select the best and execute it fully.

🔄 Critique & Refine (Constitutional)

ADVANCED

Have the model critique its own output against defined criteria, then rewrite. Dramatically improves quality without human review.

# Step 1: Generate Write a product description for {{product}}. # Step 2: Critique Review the above. Check for: - Clarity (is it jargon-free?) - Persuasion (does it sell benefits?) - Length (under 100 words?) List specific issues. # Step 3: Rewrite Rewrite addressing all issues above.
CRITIQUE CRITERIA TEMPLATES
📝 Writing: tone, clarity, brevity, voice
💻 Code: correctness, edge cases, performance, style
📊 Analysis: evidence, assumptions, bias, completeness
🎯 Strategy: feasibility, risks, alternatives, alignment

🎪 Persona Stacking

INTERMEDIATE

Assign competing expert personas to debate a topic, then synthesize.

As a VC investor: evaluate this idea. As a skeptical user: poke holes. As a product manager: find middle ground. Now synthesize all three views.

🔀 Output Anchoring

INTERMEDIATE

Start the assistant's response yourself to constrain the format from the first token.

# In API: add assistant turn start user: Analyze this startup pitch. assistant: STRENGTHS: # The model continues from there

📐 Meta-Prompting

ADVANCED

Ask the model to generate or improve the prompt itself before answering.

First, rewrite my prompt below to be clearer and more specific: """{{my_rough_prompt}}""" Then answer using the improved version.

🤖 Agent Prompting Patterns
Design patterns for autonomous AI systems that plan, act, observe, and self-correct.

⚡ ReAct — Reason + Act

ADVANCEDAGENT

Interleave reasoning (Thought) and tool execution (Action) in a loop. The gold standard for tool-using agents.

You are an agent. For each step: Thought: What do I need to do? Action: [tool_name](params) Observation: [result from tool] Repeat until task is complete, then: Final Answer: [your conclusion] # Available tools: search(query) → web results calculator(expr) → number read_file(path) → contents
ReAct TRACE EXAMPLE
Thought: I need to find the CEO of OpenAI
Action: search("OpenAI CEO 2024")
Observation: Sam Altman is CEO of OpenAI
Thought: Now I need his background
Action: search("Sam Altman background")
Observation: Former YC president, startup...
Final Answer: Sam Altman, former YC president...

🕸️ Multi-Agent Orchestration

ADVANCEDAGENT

Route tasks to specialized sub-agents. An orchestrator coordinates; specialists execute. Scales complex workflows.

🎯 ORCHESTRATOR
↓ routes tasks ↓

🔍 Researcher

Web search, retrieval

✍️ Writer

Drafting, editing

✅ Critic

Review, validate

# Orchestrator system prompt You are the Orchestrator. Your job is to: 1. Decompose the user's goal into subtasks 2. Assign each subtask to the right agent: - @researcher: information gathering - @writer: content creation - @critic: quality review 3. Synthesize agent outputs 4. Return final deliverable Always output: PLAN: [numbered subtask list] ASSIGN: [agent → subtask mapping] SYNTHESIZE: [final combined output]

🔁 Self-Reflection Agent

ADVANCEDAGENT

Agent evaluates its own output quality against rubric before returning.

After completing the task: 1. Score your output (1-10) on: - Accuracy, Completeness, Clarity 2. If any score < 7: REVISE 3. Explain what you improved 4. Return: final output + scores

🗺️ Plan-and-Execute

ADVANCEDAGENT

Separate planning from execution. Plan first, then run each step sequentially.

PHASE 1 — PLAN: List all steps to accomplish: {{goal}} Format: numbered checklist PHASE 2 — EXECUTE: Work through each step. After each: mark ✓ and note output PHASE 3 — VERIFY: Did each step succeed? If not, retry.

🧱 Scaffolded Decomposition

ADVANCEDAGENT

Break complex tasks into atomic subtasks with defined interfaces between them.

Decompose this task into atomic steps where each step: - Has a clear single input - Has a clear single output - Can be verified independently Input: {{previous_step_output}} Task: {{current_step}} Output: {{structured_result}}

💾 Memory-Augmented Agent

ADVANCEDAGENT

Maintain short-term (session) and long-term (retrieved) memory across interactions.

MEMORY CONTEXT: Short-term: {{recent_conversation}} Long-term: {{retrieved_facts}} User prefs: {{stored_preferences}} # At end of each turn: MEMORY UPDATE: New facts: [extract and store] Outdated: [flag for removal]

Critical guardrails every production agent needs:

🚫

Approval Gates

"Before executing any destructive action (delete, send, pay), confirm with user."

🔢

Iteration Limits

"If you have not completed the task in 10 steps, stop and report status."

🎯

Scope Bounding

"You may only access files in /workspace/. Never access system directories."

✅ Prompt Quality Checklist
Run your prompts through this checklist before deploying. Click to mark items done.

🌱 Beginner Checklist

Essential before any prompt goes live.

Task is stated clearly with an imperative verb (Write, Analyze, List...)
Output format is specified (JSON, bullet list, paragraph, table...)
Target audience is defined (expert, beginner, executive...)
Length/scope is bounded (word count, number of items, depth)
I've tested the prompt at least 3 times with varied inputs
Ambiguous terms are defined or removed
No conflicting instructions exist in the same prompt

🔧 Intermediate Checklist

For prompts in production or team use.

Role/persona is specific with domain expertise and communication style
At least 2 examples (input → output pairs) are provided for complex tasks
Edge cases and failure modes are handled with explicit instructions
Dynamic variables are templated with {{placeholder}} notation
Delimiters separate instructions from user-provided data
Output has been validated against a rubric or test cases
Temperature/parameters are tuned for the task type

🔴 Advanced Checklist

For complex pipelines, agents, and AI products.

System prompt is versioned and stored in source control
Prompt is tested with adversarial inputs (prompt injection attempts)
Self-critique or output validation step is included for quality assurance
Fallback behavior is defined for unexpected/out-of-scope inputs
Token usage is optimized (no redundant context)
Agent loops have iteration limits and escape conditions
Output parsing handles malformed model responses gracefully
Evals / automated tests exist for regression detection

🤖 Agent Safety Checklist

Before deploying any autonomous agent to production.

Agent scope is clearly bounded (which systems/actions it can access)
Destructive actions (delete, send, pay) require explicit human approval
Max iteration/step limit is enforced at the infrastructure level
All tool calls and outputs are logged for audit
Agent has a "report and stop" mode for unexpected situations
Sensitive data is not passed into agent context unnecessarily
Human-in-the-loop checkpoints tested and verified working
